=== IMPORTANT: DNS RECORDS ===

You MUST configure the following DNS records for mail to work correctly.
{{- range $domain, $cfg := .Values.domains }}

--- {{ $domain }} ---
  MX:      {{ $domain }}. -> {{ $.Values.hostname }}. (priority 10)
  SPF:     {{ $domain }}. TXT "v=spf1 ip4:{{ $.Values.staticPublicIPs.ipv4 }} ~all"
  DMARC:   _dmarc.{{ $domain }}. TXT "v=DMARC1; p=reject; rua=mailto:{{ default "dmarcreports" (($cfg.dmarc).localpart) }}@{{ $domain }}"
  MTA-STS: _mta-sts.{{ $domain }}. TXT "v=STSv1; id=<unique-policy-id>"
  TLSRPT:  _smtp._tls.{{ $domain }}. TXT "v=TLSRPTv1; rua=mailto:{{ default "tlsreports" (($cfg.tlsrpt).localpart) }}@{{ $domain }}"
{{- end }}

  ReverseFqdn: {{ $ipConfig.reverseFqdn | default .Values.hostname }}
{{- else }}

You must configure reverse DNS (PTR) on your public IP for mail deliverability.
For Azure, run:
  az network public-ip update \
    --resource-group <RESOURCE_GROUP> \
    --name <PUBLIC_IP_NAME> \
    --reverse-fqdn {{ .Values.hostname }}. \
    --dns-name <DOMAIN_NAME_LABEL>

NOTE: Forward DNS (A record) for {{ .Values.hostname }} must resolve to the
public IP BEFORE Azure will accept the reverse FQDN setting.
Azure only supports IPv4 reverse DNS.
{{- end }}
